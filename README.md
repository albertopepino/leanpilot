## Quick Start (5 minutes)

### Requirements

- [Docker](https://docs.docker.com/get-docker/) with Docker Compose v2
- 2 GB RAM minimum, 4 GB recommended
- Linux, macOS, or Windows (with WSL2)

### Install

```bash
# 1. Extract the package
unzip leanpilot-v1.0.zip
cd leanpilot

# 2. Run the setup script (generates secrets, builds everything, seeds DB)
chmod +x setup.sh
./setup.sh
```

That's it. Open **http://localhost:3000** and log in with the credentials shown in the terminal.

### Manual Setup (if you prefer)

```bash
# 1. Create your configuration
cp .env.example .env
# Edit .env — set DB_PASSWORD, SECRET_KEY, ADMIN_PASSWORD

# 2. Build and start
docker compose -f docker-compose.prod.yml up -d --build

# 3. Wait for services, then seed the database
docker compose -f docker-compose.prod.yml exec backend python seed_db.py
```

## Configuration

All settings are in `.env` (created from `.env.example`):

| Variable | Required | Description |
|----------|----------|-------------|
| `DB_PASSWORD` | Yes | PostgreSQL password |
| `SECRET_KEY` | Yes | JWT signing key (auto-generated by setup.sh) |
| `ADMIN_EMAIL` | No | Admin login email (default: admin@leanpilot.io) |
| `ADMIN_PASSWORD` | No | Admin password (auto-generated if empty) |
| `FACTORY_NAME` | No | Your factory name (default: My Factory) |
| `OPENAI_API_KEY` | No | Enables AI Copilot (GPT-4o). Leave empty to disable |
| `AI_MODEL` | No | OpenAI model (default: gpt-4o) |
| `APP_URL` | No | Public URL (default: http://localhost:3000) |
| `CORS_ORIGINS` | No | Allowed origins (default: http://localhost:3000) |
| `SMTP_HOST` | No | Email server for notifications |
| `SMTP_PORT` | No | SMTP port (default: 587) |
| `SMTP_USER` | No | SMTP username |
| `SMTP_PASSWORD` | No | SMTP password |

## Production Deployment (with SSL)

For deploying on a server with a domain name:

```bash
# 1. Edit .env
APP_URL=https://lean.yourdomain.com
CORS_ORIGINS=https://lean.yourdomain.com

# 2. Update nginx.conf with your domain

# 3. Start with production compose (includes nginx + certbot)
docker compose -f docker-compose.prod.yml up -d --build

# 4. Get SSL certificate
docker compose -f docker-compose.prod.yml run certbot certonly \
  --webroot -w /var/www/certbot \
  -d lean.yourdomain.com \
  --email your@email.com --agree-tos

# 5. Restart nginx to pick up the certificate
docker compose -f docker-compose.prod.yml restart nginx
```

## Architecture

```
┌──────────────────────────────────────────────┐
│              Nginx (reverse proxy)            │
│              SSL termination                  │
├──────────────┬───────────────────────────────┤
│  Next.js     │  FastAPI Backend              │
│  Frontend    │  REST API + WebSocket         │
│  :3000       │  :8000                        │
├──────────────┴───────────────────────────────┤
│  PostgreSQL 15        │  Redis 7             │
│  Persistent data      │  Sessions & cache    │
└───────────────────────┴──────────────────────┘
```

**Stack**: Next.js 14 + TypeScript + Tailwind CSS | FastAPI + SQLAlchemy + Alembic | PostgreSQL 15 | Redis 7 | Docker Compose

## Management Commands

```bash
# View logs
docker compose -f docker-compose.prod.yml logs -f

# Stop
docker compose -f docker-compose.prod.yml down

# Stop and remove all data
docker compose -f docker-compose.prod.yml down -v

# Restart a single service
docker compose -f docker-compose.prod.yml restart backend

# Access the database
docker compose -f docker-compose.prod.yml exec db psql -U leanpilot

# Re-seed (only runs if admin doesn't exist yet)
docker compose -f docker-compose.prod.yml exec backend python seed_db.py
```
